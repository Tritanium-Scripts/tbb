1. Detaillierte Beschreibung der Datenbankstruktur

  • Tabelle avatars:
    • avatar_id: Die ID des Avatars
    • avatar_address: Die Adresse zur Avatargrafik. Kann eine komplette http://-Adresse sein, aber auch ein relativer Pfad
  • Tabelle cats:
    • cat_id: Die ID der Kategorie
    • cat_l: Die ID der der im Nested-Set-Modell vorangehenden Kategorie
    • cat_r: Die ID der der im Nested-Set-Modell folgenden Kategorie
    • cat_standard_status: Legt fest, ob die Kategorie standardmäßig auf- oder zugeklappt sein soll
      0 = zugeklappt
      1 = aufgeklappt
  • Tabelle config:
    • config_name: Die Bezeichnung für eine Kofigurationsoption
    • config_value: Der Wert einer Konfigurationsoption
  • Tabelle forums:
    • forum_id: Die ID des Forums
    • cat_id: Die ID der Kategorie, der das Forum zugeordnet ist
      0 = Keine Kategorie/Forum wird nicht abgezeigt
    • order_id: Eine Zahl, die die Reihenfolge der Foren festlegt. Wer eine kleinere hat, wird früher angezeigt
    • forum_name: Der Name des Forums, der überall angezeigt wird
    • forum_description: Die Beschreibung des Forums, also ein Text, der hauptsächlich in der Forenübersicht unter einem Forum angezeigt wird
    • forum_topics_counter: Die Anzahl der Themen in einem Forum
    • forum_posts_counter: Die Anzahl der Beiträge in einem Forum
    • forum_last_post_id: Die ID des neuesten Beitrages in einem Forum
    • forum_enable_bbcode: Legt fest, ob in einem Forum BB-Code erlaubt sein soll
      0 = BB-Code deaktiviert
      1 = BB-Code aktiviert
    • forum_enable_htmlcode: Legt fest, ob in einem Forum HTML-Code erlaubt sein soll
      0 = HTML-Code deaktiviert
      1 = HTML-Code aktiviert
    • forum_enable_smilies: Legt fest, ob in einem Forum Smilies erlaubt sein sollen
      0 = Smilies deaktiviert
      1 = Smilies aktiviert
    • forum_is_moderated: Legt fest, ob ein Forum moderiert wird
      0 = Forum wird nicht moderiert
      1 = Neue Themen in einem Forum müssen freigeschaltet werden, aber keine Beiträge
      2 = Neue Themen und Beiträge müssen freigschaltet werden
    • forum_show_latest_posts: Legt fest, ob Beiträge eines Forums unter "Neueste Beiträge" erscheinen sollen
      0 = Beiträge werden nicht angezeigt
      1 = Beiträge werden angezeigt
    • auth_members_view_forum: Legt fest, ob Mitglieder ein Forum betreten dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_members_post_topic: Legt fest, ob Mitglieder Themen in einem Forum erstellen dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_members_post_reply: Legt fest, ob Mitglieder Antworten in einem Forum erstellen dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_members_post_poll: Legt fest, ob Mitglieder Umfragen in einem Forum erstellen dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_members_edit_posts: Legt fest, ob Mitglieder ihre Beitr¨ge in einem Forum bearbeiten dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_guests_view_forum: Legt fest, ob Gäste das Forum betreten dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_guests_post_topic: Legt fest, ob Gäste Themen in einem Forum erstellen dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_guests_post_reply: Legt fest, ob Gäste Antworten in einem Forum erstellen dürfen
      0 = nicht erlaubt
      1 = erlaubt
    • auth_guests_post_poll: Legt fest, ob Gäste Umfragen in einem Forum erstellen dürfen
      0 = nicht erlaubt
      1 = erlaubt
  • Tabelle forums_auth:
    • forum_id: Die ID des Forums
    • auth_type: Legt fest, ob es sich um ein Gruppenrecht oder ein Benutzerrecht handelt
      0 = Benutzerrecht
      1 = Gruppenrecht
    • auth_id: Die ID des Benutzers oder entsprechend die ID der Gruppe
    • auth_is_mod: Legt fest, ob der Benutzer oder die Gruppe Moderator des Forums ist
      0 = ist kein Moderator
      1 = ist Moderator
  • Tabelle groups:
    • group_id: Die ID der Gruppe
    • group_name: Der Name/die Bezeichnung der Gruppe
  • Tabelle groups_members:
    • group_id: Die ID der Gruppe
    • member_id: Die User-ID des Mitglieds
  • Tabelle users_locks:
    • user_id: Die ID des Users, für den die Sperre gilt
    • lock_type: Die Art der Sperren.
      0 = User hat sich registriert und muss erst freigeschaltet werden
      1 = User darf sich nicht mehr einloggen
      2 = User darf sich noch einloggen, aber keine Beiträge mehr verfassen
    • lock_start_time: Die Zeit der Erschaffung der Sperre im Unix Timestamp Format
    • lock_dur_time: Die Dauer der Sperre in Sekunden
